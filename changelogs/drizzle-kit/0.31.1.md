### Fixed `drizzle-kit pull` bugs when using Gel extensions.

Because Gel extensions create schema names containing `::` (for example, `ext::auth`), Drizzle previously handled these names incorrectly. Starting with this release, you can use Gel extensions without any problems. Hereâ€™s what you should do:

1. Enable extensions schemas in `drizzle.config.ts`

```ts
import  { defineConfig } from "drizzle-kit";

export default defineConfig({
  dialect: 'gel',
  schemaFilter: ['ext::auth', 'public']
});
```

2. Run `drizzle-kit pull`

3. Done!
